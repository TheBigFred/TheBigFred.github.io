Droits d'accès

Posté le Lun 29 avril 2019 dans linux

Il y a trois notions qui entrent en jeu pour gérer les droits d'accès d'un fichier ou d'un dossier sous UNIX, il s'agit des notions : user, group et droits.

Notion de user

La première est la notion de user (utilisateur), chaque personne travaillant sur une machine UNIX est identifiée par son user (login).

Si vous vous connectez avec le login "fred", vous serez donc l'utilisateur "fred" pour le système.

Chaque utilisateur est déclaré dans le fichier /etc/password. C'est un fichier au format texte dans lequel chaque ligne représente un utilisateur, les différentes valeurs étant séparées par :.

$ cat /etc/password 
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/bin/false
daemon:x:2:2:daemon:/sbin:/bin/false
mail:x:8:12:mail:/var/spool/mail:/bin/false
ftp:x:14:11:ftp:/srv/ftp:/bin/false
http:x:33:33:http:/srv/http:/bin/false
nobody:x:99:99:nobody:/:/bin/false
...
fred:x:1000:1000::/home/fred:/bin/bash
hugo:x:1001:1001::/home/hugo:/bin/bash
tom:x:1002:1002::/home/tom:/bin/bash
...

Voici dans l'ordre, la signification des 7 champs.

  • La première colonne correspond au login.
  • Le x indique que le mot de passe de l'utilisateur est stocké dans le fichier /etc/shadow.
  • Le premier nombre correspond au userid, c'est l'identifiant de l'utilisateur au niveau du système.
  • Le second nombre correspond au groupid, c'est l'identifiant du groupe principale de l'utilisateur au niveau du système (la notion de groupe est expliquée ci-dessous).
  • La 5ieme valeur est facultative, elle permet d'indiquer une information tel que le nom complet de l'utilisateur ou son adresse émail.
  • Le 6ieme champ indique l'emplacement du dossier personnel de l'utilisateur. C'est le seul endroit ou il pourra stocker librement ces fichiers.
  • Le dernier champ permet d'indiquer une commande ou un shell à lancer.

Pour les utilisateurs normaux on indique généralement le shell BASH, mais il peut également s'agir d'un autre shell comme Csh ou Zsh.

Certains utilisateurs peuvent être des programmes. Afin de ne pas pouvoir se connecter sous l'identifiant d'un programme, on indique la commande /bin/false qui invalide la procédure de connexion.

Pourquoi créer des utilisateurs fictifs pour des programmes ?

Lorsqu'un programme est lancé automatiquement au démarrage de la machine, il s'exécute avec l'identité de l'administrateur (nommé root sous UNIX).

Par exemple, pour un serveur web, s'il interprète du code Php, ce code est exécute sous l'identité de l'administrateur. Cela signifie que vous pouvez faire tout et n'importe quoi sur ce serveur (vous avez les droits admin) !

Pour cette raison, les programmes serveurs doivent être configurés pour s'exécuter avec un utilisateur fictif. Par fictif, on entend : si http est le nom d'utilisateur attribué au serveur web, il n'est pas possible de ce logger à la machine avec ce user.

Notion de groupe

La seconde notion est la notion de groupe (group en anglais). Chaque utilisateur d'une machine UNIX est repris dans un ou plusieurs groupes d'utilisateurs.

Si l'on donne un droit à un groupe, tous les utilisateurs faisant partie de ce groupe possèdent ce droit.

En général, lorsque l'administrateur de la machine UNIX crée un utilisateur, il crée également un groupe portant le même nom et le même userid. Il s'agit de votre groupe par défaut.

Certains groupes représentent des programmes, d'autres des ressources (carte vidéo, la carte son, lecteur CD, ...).

Chaque groupe est déclaré dans le fichier /etc/group. C'est un fichier au format texte dans lequel chaque ligne représente un groupe, les différentes valeurs étant séparées par :.

$ cat /etc/group
root:x:0:root
bin:x:1:root,bin,daemon
daemon:x:2:root,bin,daemon
...
video:x:91:fred,hugo,tom
users:x:100:fred,hugo,tom
...
fred:x:1000:
hugo:x:1001:
tom:x:1002:
...

Voici dans l'ordre la signification des 4 champs.

  • La première colonne correspond au nom du groupe.
  • Le x représente le mot de passe du groupe, mais n'est pas utilisé.
  • La troisième colonne est le groupid.
  • Le dernier champ est la liste des utilisateurs faisant partie du groupe. Chaque login est séparé par une virgule.

Attribution du user/group

A chaque fichier (ou dossier) est assigné un utilisateur (le propriétaire) et un groupe.

Plus exactement, le user/group attribué à un fichier (ou dossier) est le userid et le groupid effectif de l'application ou du user qui le créé.

Lorsque vous créez un fichier (ou un dossier), vous en êtes automatiquement le propriétaire. Le groupe assigné au fichier (ou dossier) sera votre groupe par défaut, qui porte généralement le même nom que votre nom d'utilisateur.

Voici un exemple de fichiers et dossiers :

$ ls -l
...      fred   fred   fichier1.txt
...      fred   fred   dossier1    
...      fred   user   fichier2.txt
...      fred   user   dossier2    
...      tom    tom    fichier3.txt
...      tom    users  dossier3

Si vous êtes le propriétaire d'un fichier (ou dossier), vous pouvez changer le groupe de ce fichier (ou dossier).

Si vous êtes le super administrateur (root), vous pouvez changer le propriétaire et le groupe de ce que vous voulez.

Notion de droits

Il y a trois types de droits :

  • le droit de lecture, noté "r":

    • dans le cas d'un fichier, il permet de lire le fichier ;
    • dans le cas d'un dossier, il permet de lister le dossier ;

  • le droit d'écriture, noté "w":

    • dans le cas d'un fichier, il permet de modifier/supprimer le fichier ;
    • dans le cas d'un dossier, il permet d'écrire dans le dossier ( créer des sous-dossiers, des fichiers, ... ),
      si vous avez le droit de supprimer touts les fichiers et sous-dossiers contenus, vous pouvez supprimer ce dossier.

  • le droit d'exécution, noté "x":

    • dans le cas d'un fichier, il permet de d'exécuter le fichier ( qui doit évidement être un programme, sinon cela n'a pas de sens ) ;
    • dans le cas d'un dossier, il permet d'entrer dans le dossier.

Voici un exemple de fichiers et dossiers :

$ ls -l
rw- rw- r--   fred   fred   fichier1.txt
rwx --- ---   fred   fred   dossier1    
rw- --- r--   fred   user   fichier2.txt
r-x r-x r-x   fred   user   dossier2    
r-- r-- r--   tom    tom    fichier3.txt
rwx --- r-x   tom    users  dossier3

Chaque fichier (ou dossier) possède des droits rwx pour le propriétaire, rwx pour le groupe attribué au fichier (ou dossier) et rwx pour les autres (ceux qui ne font pas partie du groupe du fichier). Les droits sont toujours donnés dans l'ordre rwx.

Dans l'exemple ci avant, pour le fichier le fichier1.txt , le propriétaire peut lire et écrire , les utilisateurs du groupe peuvent également lire et écrire dans le fichier, les autres utilisateurs du système ne peuvent que lire le fichier. Le - montre l'absence d'un droit.

Valeurs numériques des droits

Les droits rwx correspondent à un champ de 3 bits. Le bit vaut 1 si le droit est donné, sinon il vaut zéro. Le tableau suivant illustre les valeurs possibles et l'équivalent numérique.

droits valeur en binaire valeur en décimal
- - - 0 0 0 0
- - x 0 0 1 1
- w - 0 1 0 2
r - - 1 0 0 4
r w x 1 1 1 7
r w - 1 1 0 6
r - x 1 0 1 5
- w x 0 1 1 3

Partager un dossier

Prenons comme exemple le dossier /home qui contient les dossiers des utilisateurs :

$ ls -l /home
rwx --- ---  fred fred /home/fred
rwx --- ---  tom  tom  /home/tom

Ici seul l'utilisateur fred peut accéder au dossier /home/fred, de même que tom ne peut accéder qu'à son dossier.

Maintenant, nous allons chercher à partager un sous-dossier de fred avec tom.

1ier méthode

Une première manière de procéder est de donner un droit de lecture et d'exécution au groupe fred et de mettre tom dans le groupe fred. Les droits deviennent ceci :

$ chmod 750 /home/fred
$ ls -l /home
rwx r-x ---  fred fred /home/fred
rwx --- ---  tom  tom  /home/tom

Le problème est que l'administrateur n'ajoutera pas tom dans votre groupe ( il n'a pas que cela à faire ). De plus, si votre groupe contient 40 utilisateurs, Ces 40 utilisateurs peuvent accéder au dossier personnel de fred.

2ieme méthode

Une deuxième manière de faire est de changer le groupe du fichier en tom, les droits deviennent ceci :

$ chown fred:tom /home/fred
$ ls -l /home
rwx r-x ---  fred tom  /home/fred
rwx --- ---  tom  tom  /home/tom

Cette méthode à pour inconvénient que tom et tous les membres du groupe tom peuvent accéder au contenu du dossier personnel de fred.

Si l'on souhaite partager un autre sous-dossier de /home/fred avec l'utilisateur hugo. On ne sait pas, on ne peut indiquer qu'un seul nom de groupe par fichier et dossier.

3ieme méthode

L'astuce consiste à donner le droit --x à tout le monde sur le dossier /home/fred.

De cette manière, tout le monde à le droit d'entrer dans le dossier (cd /home_fred), mais pas de lister le contenu du dossier (ls /home/fred).

$ chown fred:fred /home/fred
$ chmod 701 /home/fred
$ ls -l /home
rwx --- --x  fred tom  /home/fred
rwx --- ---  tom  tom  /home/tom

Attention, les autres ne savent pas lister le dossier, mais rien n'empêche d’essayer au hasard d’entrer dans un sous-dossier.

Regardons le contenu du dossier de fred (en tant que fred)

$ ls -l /home/fred
rwx rwx rwx fred fred   top_secret
rwx rwx rwx fred fred   share_tom
rw- rw- rw- fred fred   carte_visa.txt

Ici, tout est en accès libre. Les autres utilisateurs ne voient pas vos fichiers, mais s'ils tapent des noms de fichiers (ou de dossiers) au hasard, il est possible qu'ils ouvrent ce qu'ils ne doivent pas.

On doit donc régler les droits dans /home/fred comme ceci:

$ chmod 770 *
$ ls -l /home/fred
rwx rwx --- fred fred   top_secret
rwx rwx --- fred fred   share_tom
rwx rwx --- fred fred   carte_visa.txt

Le but est que tom puisse aller lire dans le dossier share_tom, on doit donc adapter le groupe et les droits de ce dossier :

$ chmod 750 share_tom
$ chown fred:tom share_tom
$ ls -l /home/fred
rwx rwx --- fred fred   top_secret
rwx r-x --- fred tom    share_tom
rwx rwx --- fred fred   carte_visa.txt

Attention, tous les membres du groupe tom peuvent accéder au sous-dossier share_tom. Il est évident que tom doit connaître le nom du dossier que je lui destine.

Droits spéciaux

Un processus est un programme en cours d’exécution. Le fichier binaire qui représente ce programme appartient, comme tous fichiers, à un user et à un groupe. C'est ce que l'on nomme real user id (ruid) et real group id (rgid).

Le ruid et rgid ont généralement la valeur root puisque c'est lui qui installe les logiciels sur le système. Vos programmes personnels, stockés dans /home/... auront un ruid et un rgid correspondant à votre user.

Chaque processus possède également un effective user id (euid) et effective group id (egid). Il s'agit du user et du groupe avec lesquels s'exécute le processus. Un processus exécuté par root aura un euid=0 et un egid=0.

Aux 9 bits des droits classiques, on associe 2 autres bits qui agissent sur les programmes exécutables. Il s'agit de :

  • SETUID bit => valeur décimal 4 ;
  • SETGID bit => valeur décimal 2 ;

Il y a un 3ieme bit qui agit sur les dossiers :

  • STICKY bit => valeur décimal 1.

Le sticky bit d'un répertoire, définit que seul le propriétaire peut supprimer les fichiers de ce dossiers. Même si d'autres utilisateurs disposent d'un droit en écriture sur ce répertoire. Cela permet de transformer un dossier en une boite aux lettres.

Lorsque l'on fait un ls -l, on aura par exemple : rwx rwx rwt t indique que le sticky bit est positionné.

Pour illustrer setuid et setgid, j'ai écrit un petit programme qui crée un fichier hello.txt dans le dossier /root.

#include <stdio.h>

int main()
{
    FILE* fp = fopen("/root/hello.txt","w");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }

    fprintf(fp,"Hello world!!!");
    fclose(fp);
    return 0;
}

Compilons et exécutons une première fois ce programme.

$ gcc -o hello hello.c
$ ls -l hello
rwx rwx r-x fred fred  hello
$ ./hello 
fopen: Permission denied

Positionnons le setuid.

$ sudo chown root:root hello
$ sudo chmod 4755 hello
$ ls -l hello
rws r-x r-x root:root  hello
$ sudo ls /root
$ ./hello
$ sudo ls /root
hello.txt

Tout utilisateur qui exécute le binaire les mêmes droits que le propriétaire du fichier (ici root).

Si on positionne le setgid (chmod 2755), tout utilisateur exécute le binaire avec les mêmes droits que le goup du fichier (ici root). La commande ls -l hello donnera : rwx r-s r-x root root. hello
